# SpatialMind Skills 开发规范

## 一、Skills 通用规范

### 1.1 定义

Skills 是一组调用 LLM 增强输出的模块，位于 `skills/` 目录，分为两个 Skill 包：
- **NaturePublishSkill**: 发表级图表与文字输出
- **BioInsightSkill**: 生物学洞察与解读

### 1.2 通用规则

1. **必须标注免责声明**: 所有 AI 生成的解读类输出必须在开头标注 `"AI 生成内容，仅供参考"` 或 `"AI-generated content is for reference only"`
2. **不编造数值**: 只能引用 `step_results` 中的真实数据指标，不能编造 p-value、百分比、基因名
3. **函数签名**: 每个 Skill 函数接收 `AgentState` 作为输入，返回可序列化的 dict
4. **可选的 LLM 调用**: Skills 可以调用 LLM，但必须通过 `agent/llm_client.py`
5. **异常处理**: 所有 LLM 调用必须 try/except，失败时返回错误信息而非崩溃

### 1.3 与 Tools 层的区别

| 维度 | Tools 层 | Skills 层 |
|------|---------|----------|
| LLM 调用 | ❌ 禁止 | ✅ 允许 |
| 数据修改 | ✅ 修改 adata | ❌ 只读不修改 |
| 输出类型 | metrics + 图表 | 文字 + 报告 |
| 调用时机 | executor 节点 | skill_invoker 节点 |

## 二、NaturePublishSkill

### 2.1 概述

将空间转录组分析结果包装成符合 Nature/Cell/Science 期刊投稿规范的输出。

### 2.2 模块接口

#### palette.py — 配色方案
```python
def get_palette(journal: str = "nature", n_colors: int = 10) -> list[str]
```
- `journal`: "nature" | "cell" | "science"
- 返回十六进制颜色代码列表

#### panel_composer.py — 多子图排版
```python
def compose_panels(state: AgentState) -> dict
```
- 将 `figures` 中的独立图片组合成 Figure 1 风格的多子图
- 返回 `{"panel_path": str, "n_panels": int}`

#### caption_writer.py — 图注生成
```python
def write_captions(state: AgentState) -> str
```
- 调用 LLM，基于 `step_results` 生成英文图注
- 格式: "Figure X: ..." 的学术规范

#### methods_writer.py — Methods 段落生成
```python
def write_methods(state: AgentState) -> str
```
- 调用 LLM，基于实际使用的参数生成 Methods 段落
- 引用 Scanpy/Squidpy 版本号

#### language_polish.py — 语言润色
```python
def polish_text(text: str) -> str
def polish_all(state: AgentState) -> dict
```
- 调用 LLM，将分析摘要润色为学术英文

#### cover_letter.py — Cover Letter 生成
```python
def generate_cover_letter(state: AgentState) -> str
```
- 调用 LLM，生成标准格式的投稿信

#### word_exporter.py — Word 导出
```python
def export_word(state: AgentState) -> dict
```
- 使用 python-docx 生成 .docx 报告文件
- 包含: 标题→分析步骤→图表→Methods→图注

#### html_report.py — HTML 报告
```python
def generate_html_report(state: AgentState) -> str
```
- 生成独立可分享的 HTML 文件
- 图表以 base64 内嵌
- 响应式设计，带 CSS 样式

### 2.3 NaturePublishSkill 输出格式

```json
{
    "palette": [...],
    "panel_paths": [...],
    "captions": "...",
    "methods": "...",
    "cover_letter": "...",
    "docx_path": "...",
    "html_path": "...",
    "disclaimer": "AI-generated content..."
}
```

## 三、BioInsightSkill

### 3.1 概述

提供超越数据本身的生物学洞察，帮助研究人员理解分析结果的含义。

### 3.2 模块接口

#### cluster_namer.py — Cluster 命名
```python
def name_clusters(state: AgentState) -> dict
```
- 从 `adata.uns["rank_genes_groups"]` 提取 marker genes
- 调用 LLM 推断细胞类型
- 返回 `{"cluster_names": {...}, "disclaimer": "..."}`

#### spatial_story.py — 空间叙事
```python
def generate_spatial_story(state: AgentState) -> str
```
- 基于空间分布和聚类结果，生成可读的空间模式描述
- 只描述空间模式，不具功能结论

#### qc_interpreter.py — QC 解读
```python
def interpret_qc(state: AgentState) -> str
```
- 判断 QC 指标是否在正常范围
- 给出可操作的建议

#### next_step.py — 下一步建议
```python
def suggest_next_steps(state: AgentState) -> str
```
- 基于已完成的分析推荐 1-3 个后续方向
- 推荐: SVG / CellChat / GO enrichment / trajectory / integration

#### lit_reader.py — 文献阅读
```python
def read_pdf(file_path: str, state: AgentState) -> str
```
- 使用 pypdf 提取 PDF 文本
- 调用 LLM 总结并与当前分析关联

#### insight_extractor.py — 关键发现
```python
def extract_insights(state: AgentState) -> str
```
- 从所有 step_results 中提炼 3-5 条关键发现
- 每条发现必须有数据支持

### 3.3 BioInsightSkill 输出格式

```json
{
    "cluster_names": "...",
    "spatial_story": "...",
    "qc_interpretation": "...",
    "next_steps": "...",
    "literature_summary": "...",
    "insights": "...",
    "disclaimer": "AI 生成内容，仅供参考..."
}
```

## 五、外部库集成说明

### 5.1 ClawBio（生信通用 Agent 技能库）

- **位置**: `D:\SEU\AI tech\ClawBio`
- **项目**: https://github.com/ClawBio/ClawBio — 第一代生物信息学原生 AI Agent 技能库，88 个技能（29 个 production-ready）+ 8000+ Galaxy 工具
- **版本**: v0.5.2（`pip install clawbio`）
- **核心接口**: `from clawbio import run_skill, list_skills`
- **与本项目 Skills 的交叉**:

| ClawBio 技能 | 功能 | 是否可替代本项目的模块 |
|-------------|------|---------------------|
| `scrna-orchestrator` | scRNA-seq 全流程编排（QC → 降维 → 聚类 → 标注） | ⚠️ 部分重叠，但 SpatialMind 专为空间转录组设计，建议不直接替代 |
| `sample-qc-triage` | QC 指标评估与决策（判断是否通过 QC） | ⚠️ 思路可借鉴，但需适配空间数据特征 |
| `bio-orchestrator` | 通用生信步骤编排引擎 | 🔧 可借鉴其 `orchestrator.py` 中的步骤调度逻辑 |
| `bioqc-mcp` | MCP 协议 bioQC 服务 | 🔧 可作为 MCP 扩展参考 |

- **建议集成方式**: 以借鉴代码思路为主。`sample-qc-triage` 的 QC 决策逻辑（自动判断 QC pass/fail 的阈值策略）和 `scrna-orchestrator` 的分析流程组织方式值得参考，但不直接替换现有 `skills/` 目录下的实现，因为 SpatialMind 需要空间坐标相关的特殊处理（Stereo-seq / Visium）。

- **位置**: `D:\SEU\AI tech\nature-skills`
- **项目**: https://github.com/Yuan1z0825/nature-skills — 开源科研技能集，90,000+ GitHub Stars
- **与本项目 Skills 的交叉**:

| nature-skills 技能 | 功能 | 是否可替代本项目的模块 |
|-------------------|------|---------------------|
| `nature-figure` | 投稿级科研绘图（Python + matplotlib/seaborn | R + ggplot2），含 figure contract、配色、导出 | ✅ 可替代或增强 `NaturePublishSkill` 的绘图部分（`palette.py` + `panel_composer.py`） |

| `nature-paper2ppt` | 论文转 PPT | 🔧 可作为 Export 能力扩展 |
| `nature-reviewer` | 审稿意见生成 | 🔧 可作为 Export 能力扩展 |
| `nature-citation` | 引用格式管理 | 🔧 可作为辅助工具 |
| `nature-response` | 审稿回复信生成 | 🔧 可作为 Export 能力扩展 |
| `nature-data` | 数据表格处理 | 🔧 可作为辅助工具 |

- **建议集成方式**:

  2. **nature-polishing**: 可直接在 `language_polish.py` 的 LLM prompt 中引用 `nature-polishing/references/` 中的句式库和写作策略，提升润色质量。
  3. **nature-writing**: 可在 `caption_writer.py`、`methods_writer.py`、`cover_letter.py` 的 prompt 中注入 nature-writing 的章节架构规则。
  4. **不建议直接替换**现有文件，而是通过修改 agent prompt 的方式增强现有实现，保持项目结构稳定。
